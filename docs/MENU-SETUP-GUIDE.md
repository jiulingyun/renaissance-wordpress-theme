# 多语言菜单设置指南

## 自动创建的内容

Renaissance 主题在激活时会自动创建以下菜单：

### 英语菜单
- **Primary Navigation** - 主导航菜单
- **Footer Navigation** - 页脚导航菜单

### 中文菜单
- **Primary Navigation - 中文** - 主导航菜单（中文）
- **Footer Navigation - 中文** - 页脚导航菜单（中文）

### 法语菜单
- **Primary Navigation - Français** - 主导航菜单（法语）
- **Footer Navigation - Français** - 页脚导航菜单（法语）

所有菜单都已经包含了正确的菜单项，并且已经在数据库中预设了菜单位置关联。

## 手动绑定菜单位置（必需步骤）

由于 Polylang 插件的设计限制，需要在后台手动确认菜单位置绑定：

### 步骤：

1. **进入菜单管理页面**
   - 登录 WordPress 后台
   - 导航到：外观 → 菜单

2. **绑定中文菜单**
   - 找到 **Primary Menu 中文 (中国)** 行
   - 在下拉框中选择 **Primary Navigation - 中文**
   - 找到 **Footer Menu 中文 (中国)** 行
   - 在下拉框中选择 **Footer Navigation - 中文**

3. **绑定法语菜单**
   - 找到 **Primary Menu Français** 行
   - 在下拉框中选择 **Primary Navigation - Français**
   - 找到 **Footer Menu Français** 行
   - 在下拉框中选择 **Footer Navigation - Français**

4. **保存更改**
   - 点击页面底部的 **保存更改** 按钮

### 完成！

现在访问网站首页，切换语言时，菜单会自动显示对应语言的内容。

## 注意事项

- ✅ 英语菜单已自动绑定，无需手动操作
- ✅ 所有菜单项都已正确创建，包括翻译版本
- ✅ 菜单项会自动链接到对应语言的页面
- ⚠️ 如果需要修改菜单项，请在对应语言的菜单中编辑

## 故障排除

### 问题：菜单显示为空白或圆点
**解决方案**：确保已经在后台绑定了菜单位置（见上述步骤）

### 问题：切换语言后菜单没有变化
**解决方案**：
1. 检查是否已为所有语言绑定菜单
2. 清除浏览器缓存
3. 清除 WordPress 缓存（如果使用了缓存插件）

### 问题：菜单项链接错误
**解决方案**：
1. 确保所有页面都已创建翻译版本
2. 在 Polylang 设置中检查页面的语言关联
3. 重新保存菜单设置

## 技术说明

主题会自动执行以下操作：
- 创建所有语言的菜单
- 添加菜单项并链接到翻译页面
- 设置菜单的 Polylang 语言属性
- 在数据库中保存菜单位置关联

但是，Polylang 的后台界面需要用户手动确认菜单位置绑定才能在管理界面中显示。这是 Polylang 插件的设计限制，不是主题的问题。
